About

This theme is a html5 template designed for admin user interface. It has client side features which is the starting point of your web application or Content Management System (CMS).

Please note that this theme is NOT a ready to use CMS. It contains a lot of client side scripts (Javascript) but didn't come with server side script (PHP/ASP/JSP etc). The purpose of this item is to provide you basic features for your own web application or CMS.

Layout Structure

This theme is a fixed/liquid layout with two columns. All of the information within the main content area is nested within a section with an id of "main". The sidebar's content is within an aside with an id of "sidebar". The general template structure is the same throughout the template. Here is the general structure.

General Structure

The general page structure is divided into 4 parts: <header>, <nav>, <section> and <footer>.

<body>
	<header id="top"></header>
	<nav id="topmenu"></nav>
	<section id="content"></section>
	<footer id="bottom"></footer>
</body>

Content Structure

The content area (<section id="content">) is divided into 2 columns: the <section id="main"> for main content and <aside id="sidebar"> for sidebar.

<section id="main">
	<article>
		<h1>Heading</h1>
		
		<h2>Heading</h2>
		<div>content here....</div>
		
		<h2>Heading</h2>
		<div>content here....</div>
	</article>
	<article>
		<h1>Heading</h1>
		
		<h2>Heading</h2>
		<section>content here....</section>
		
		<h2>Heading</h2>
		<section>content here....</section>
	</article>
</section>

<aside id="sidebar">
	<div class="box">
		<h2>Heading</h2>
		<section>
			content here....
		</section>
	</div>
	<div class="box">
		<h2>Heading</h2>
		<section>
			content here....
		</section>
	</div>
</aside>

Grid System

This theme uses grid system generated from Variable Grid System. The layout is 12-column grid.

Examples of using the grid:

2 columns:


	

		{LEFT COLUMN}
	

	

		{RIGHT COLUMN}

3 columns:


	

		{LEFT COLUMN}
	

	

		{MIDDLE COLUMN}
	

	

		{RIGHT COLUMN}

You can create up to 12 columns layout with this grid system.

For this theme I use this markup for 2 columns:


	

		{RIGHT COLUMN}
	

	

		{LEFT COLUMN}

By using the push_xx and pull_xx classes, elements can be rearranged, so the order in the markup is the right column (main content) first before the left column (sidebar).

Form

Use a form with class=uniform to create a styled form. Each form elements can be separated by using definition list elements (dl, dt, dd).
This is the basic markup for generating form:


	

		

		

			
			An input description

To make form inputs inline with the labels (input at the right of label), just add class=inline attribute in the dl:


	

		

		

			
			An input description

To add a validation in the form, add an id in the form, for example <form id="loginform">, and add this javascript code:

$(document).ready(function() {
	$('#loginform').validate();
});

You can activate datatables on any table. Just add an id on a table, for example <table id="data">, and add this javascript code:

$(document).ready(function() {
	$('#data').dataTable();
});

Collapsible boxes

The boxes in the sidebar can be toggled to show or hide their content. Click on the arrow icon at the top right of the box. The html structure:

Fixed/Fluid Layouts

The default layout for this theme is a fixed layout. You can switch from fixed to fluid layout by adding fluid class to the body.

Fixed layout:

<body>
...
</body>

Fluid layout:

<body class="fluid">
...
</body>

The fixed layout is using a fixed 960 pixels width, whereas the fluid layout is using percentage width (so the view is adjusted to the user's screen resolution).

The custom.js contains scripts for setting plugins or add custom scripts, e.g. tabs, accordion, toggle boxes, checkall checkboxes, detail table, and message boxes. This file is commented so you can easily find each sections.

To disable plugins, delete the <script src="js/plugin-name.js"> (in the html file), and the scripts in custom.js. For example: if you don't want to use the WYSIWYG editor, delete this:

<link rel="stylesheet" type="text/css" href="css/jquery.wysiwyg.css">

<script type="text/javascript" src="js/jquery.wysiwyg.js"></script>
and also delete this in custom.js: 
/* wysiwyg editor
------------------------------------------------------------------------- */
$('#wysiwyg').wysiwyg();
$('#newscontent').wysiwyg();

To view more about plugins usage, you can click the links of each plugins in Sources and Credits below.
